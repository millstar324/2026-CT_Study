# 🚀 알고리즘 레벨업 스터디 🌱

> **프로그래머스 Lv.1 ~ Lv.2 정복을 위한 코드 기록 및 리뷰 스터디입니다.**

---

## 📅 운영 룰 (Problem Solving Routine)

### 1. 문제 선정 및 출제
* **난이도:** 프로그래머스 **Lv.1 (정답률 70% 이하)** ~ **Lv.2**
* **출제 방식:** 팀원별로 순번을 정해 **매일** 한 문제씩 추천합니다.
* **폴더 생성:** 문제 출제자가 해당 문제의 폴더를 직접 생성합니다.
    * *폴더명 규칙:* `DAY[] Lv[레벨]_[문제명]` (예: `[DAY2] Lv2_소수만들기`)

### 2. 협업 프로세스 (Flow)
1. **브랜치 생성:** `main` 브랜치에서 직접 작업하지 않고, **본인 닉네임의 브랜치**를 생성합니다.
   - 예: `feature/[닉네임]`
2. **코드 업로드:** 출제자가 만든 문제 폴더 안에 본인 코드를 업로드합니다.
   - **파일명 규칙:** `[문제명]_[이름].[확장자]` (예: `소수만들기_홍길동.py`)
3. **PR 생성:** 업로드 후 `main` 브랜치 방향으로 **Pull Request**를 보냅니다.
4. **최종 머지:** 모든 팀원의 코드가 제출되면 **방장이 검토 후 `main`에 최종 머지**합니다.
   - 💡 **주의:** 방장의 머지가 완료되어야 프로필에 잔디가 확정적으로 심어집니다! 🌱
5. **생성된 닉네임 브랜치 삭제:** 코드 꼬일 일을 방지하기 위해 로컬과 홈페이지에 **본인 닉네임 브랜치**를 삭제한 뒤 마무리합니다.
